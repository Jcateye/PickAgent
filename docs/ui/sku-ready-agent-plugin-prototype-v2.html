本次调整重点

插件本质是自动化采集工具，不是配置台。因此默认界面应优先呈现“现在正在做什么、做到哪一步、下一步会做什么”，而不是让用户先看到字段配置细节。

1. 信息优先级重排

  • 把 运行进度 提升到主内容前段，成为扫描之后最重要的信息块。
  • 把 可采集字段 / 字段映射 收起，作为“有需要再看”的审查内容。
  • 风险仍保留展开态，因为它是运营最关心的结果层。

2. 节点进度轴语义

  • 已完成：绿色节点，表示该步骤已有结果。
  • 进行中：蓝色高亮节点，表示插件正在执行。
  • 等待中：灰蓝节点，表示后续队列步骤。
  • 循环下一页：单独用 loop note 展示，强调自动翻页继续采集。

3. 适配你的业务流程

解析 SKU 列表 解析 SKU XXXX 采集商品指标 重点评论采集 更新统计数据 继续下一页

4. 产品层建议

如果后续进入开发，建议把进度轴做成真实 run 流式更新组件：每个节点都带 step id、开始时间、耗时、异常状态、重试按钮和关联证据数量。这样未来无论插件端还是控制台端，都能复用为统一的 Run Timeline。
