# design.md — SKU Ready Agent

SKU Ready Agent is an operations console, not a showcase.

Optimize for fast diagnosis, traceable decisions, and safe human approval.

One page = one primary job.
Do not flatten all capabilities onto one screen.
Default views show only status, conclusion, next action, and evidence summary.
Full evidence, raw fields, rules, tool traces, history, and exports must be hidden behind drawers, tabs, collapses, or modals.
Tables are for SKU-scale work; cards are for summaries and decisions.
Chat is one console surface, not the product center.
Plugin Side Panel is a run monitor first, configuration panel second.
Every AI conclusion must link to evidence, rule, run, tool trace, or review gate.
High-risk actions are recommendations plus human approval, never silent execution.

Visual style: modern B2B SaaS, restrained, compact, light, ordered.
Avoid landing pages, giant hero sections, decorative gradients, glassmorphism, purple AI glow, and equal-weight card walls.
